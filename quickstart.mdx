# Quick Start Guide

This guide will walk you through creating your first assessment with the Skillment API. You'll learn how to:

- Register an organization and get API credentials
- Create your first coding assessment
- Add participants and send invitations
- Monitor assessment progress

<Note>
**Prerequisites**: You'll need a valid email address and basic familiarity with REST APIs and cURL commands.
</Note>

## Step 1: Register Your Organization

First, let's register your organization and create an admin account:

<CodeGroup>
```bash cURL
curl -X POST https://api.skillment.in/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@mycompany.com",
    "password": "securePassword123",
    "firstName": "John",
    "lastName": "Doe",
    "phone": "+1234567890",
    "gender": "male",
    "orgName": "my-tech-company",
    "orgType": "technology",
    "orgSize": "50-100",
    "plan": "free",
    "termsAccepted": true,
    "newsletterOptIn": false
  }'
```

```javascript JavaScript
const response = await fetch('https://api.skillment.in/api/auth/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'admin@mycompany.com',
    password: 'securePassword123',
    firstName: 'John',
    lastName: 'Doe',
    phone: '+1234567890',
    gender: 'male',
    orgName: 'my-tech-company',
    orgType: 'technology',
    orgSize: '50-100',
    plan: 'free',
    termsAccepted: true,
    newsletterOptIn: false
  })
});

const data = await response.json();
console.log('Registration successful:', data);
```

```python Python
import requests

url = "https://api.skillment.in/api/auth/register"
payload = {
    "email": "admin@mycompany.com",
    "password": "securePassword123",
    "firstName": "John",
    "lastName": "Doe",
    "phone": "+1234567890",
    "gender": "male",
    "orgName": "my-tech-company",
    "orgType": "technology",
    "orgSize": "50-100",
    "plan": "free",
    "termsAccepted": True,
    "newsletterOptIn": False
}

response = requests.post(url, json=payload)
data = response.json()
print("Registration successful:", data)
```
</CodeGroup>


<Tip>
**Save your token!** Copy the `token` from the response - you'll need it for all subsequent API calls. The token is valid for 7 days.
</Tip>

## Step 2: Create Your First Assessment

Now let's create a coding assessment for JavaScript developers:

<CodeGroup>
```bash cURL
curl -X POST https://api.skillment.in/api/assessments \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "JavaScript Developer Assessment",
    "description": "Test JavaScript fundamentals and problem-solving skills",
    "type": "coding",
    "duration": 60,
    "questions": [
      {
        "type": "coding",
        "question": "Write a function that returns the sum of two numbers",
        "difficulty": "Easy",
        "marks": 10,
        "timeLimit": 15,
        "starterCode": {
          "javascript": "function sum(a, b) {\n  // Your code here\n}",
          "python": "def sum(a, b):\n    # Your code here\n    pass"
        },
        "testCases": [
          {
            "input": "sum(2, 3)",
            "expectedOutput": "5",
            "isVisible": true
          },
          {
            "input": "sum(-1, 1)",
            "expectedOutput": "0",
            "isVisible": true
          },
          {
            "input": "sum(100, 200)",
            "expectedOutput": "300",
            "isVisible": false
          }
        ]
      }
    ],
    "settings": {
      "proctoringEnabled": true,
      "randomizeQuestions": false,
      "showResults": false
    }
  }'
```

```javascript JavaScript
const assessmentData = {
  title: "JavaScript Developer Assessment",
  description: "Test JavaScript fundamentals and problem-solving skills",
  type: "coding",
  duration: 60,
  questions: [
    {
      type: "coding",
      question: "Write a function that returns the sum of two numbers",
      difficulty: "Easy",
      marks: 10,
      timeLimit: 15,
      starterCode: {
        javascript: "function sum(a, b) {\n  // Your code here\n}",
        python: "def sum(a, b):\n    # Your code here\n    pass"
      },
      testCases: [
        {
          input: "sum(2, 3)",
          expectedOutput: "5",
          isVisible: true
        },
        {
          input: "sum(-1, 1)",
          expectedOutput: "0",
          isVisible: true
        },
        {
          input: "sum(100, 200)",
          expectedOutput: "300",
          isVisible: false
        }
      ]
    }
  ],
  settings: {
    proctoringEnabled: true,
    randomizeQuestions: false,
    showResults: false
  }
};

const response = await fetch('https://api.skillment.in/api/assessments', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(assessmentData)
});

const assessment = await response.json();
console.log('Assessment created:', assessment);
```
</CodeGroup>

## Step 3: Add Participants

Let's add some candidates who will take the assessment:

<CodeGroup>
```bash cURL (Single Participant)
curl -X POST https://api.skillment.in/api/participants \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "Alice",
    "lastName": "Johnson",
    "email": "alice.johnson@email.com",
    "phone": "+1234567890",
    "tags": ["frontend", "javascript", "react"]
  }'
```

```bash cURL (Bulk Participants)
curl -X POST https://api.skillment.in/api/participants/bulk \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "participants": [
      {
        "firstName": "Alice",
        "lastName": "Johnson",
        "email": "alice.johnson@email.com",
        "tags": ["frontend", "javascript"]
      },
      {
        "firstName": "Bob",
        "lastName": "Smith",
        "email": "bob.smith@email.com",
        "tags": ["backend", "node.js"]
      }
    ]
  }'
```

```javascript JavaScript
// Add single participant
const participantData = {
  firstName: "Alice",
  lastName: "Johnson",
  email: "alice.johnson@email.com",
  phone: "+1234567890",
  tags: ["frontend", "javascript", "react"]
};

const response = await fetch('https://api.skillment.in/api/participants', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(participantData)
});

const participant = await response.json();
console.log('Participant added:', participant);
```
</CodeGroup>

## Step 4: Assign Assessment to Participants

Now let's assign our assessment to the participants:

<CodeGroup>
```bash cURL
curl -X POST https://api.skillment.in/api/assessments/assessment_789/assign \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "participantIds": ["participant_101", "participant_102"],
    "sendInvitation": true,
    "customMessage": "Please complete this assessment by tomorrow."
  }'
```

```javascript JavaScript
const assignmentData = {
  participantIds: ["participant_101", "participant_102"],
  sendInvitation: true,
  customMessage: "Please complete this assessment by tomorrow."
};

const response = await fetch('https://api.skillment.in/api/assessments/assessment_789/assign', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(assignmentData)
});

const result = await response.json();
console.log('Assignment successful:', result);
```
</CodeGroup>

## Step 5: Send Candidate Credentials

Generate and send login credentials to candidates:

<CodeGroup>
```bash cURL
curl -X POST https://api.skillment.in/api/email/send-credentials \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "participantId": "participant_101",
    "assessmentId": "assessment_789"
  }'
```

```javascript JavaScript
const credentialsData = {
  participantId: "participant_101",
  assessmentId: "assessment_789"
};

const response = await fetch('https://api.skillment.in/api/email/send-credentials', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(credentialsData)
});

const result = await response.json();
console.log('Credentials sent:', result);
```
</CodeGroup>

## Step 6: Monitor Assessment Progress

Check the status of your assessment and participant progress:

<CodeGroup>
```bash cURL (Assessment Stats)
curl -X GET https://api.skillment.in/api/assessments/stats \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

```bash cURL (Assessment Candidates)
curl -X GET https://api.skillment.in/api/assessments/assessment_789/candidates \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

```javascript JavaScript
// Get assessment statistics
const statsResponse = await fetch('https://api.skillment.in/api/assessments/stats', {
  headers: {
    'Authorization': `Bearer ${token}`,
  }
});
const stats = await statsResponse.json();
console.log('Assessment stats:', stats);

// Get candidates for specific assessment
const candidatesResponse = await fetch('https://api.skillment.in/api/assessments/assessment_789/candidates', {
  headers: {
    'Authorization': `Bearer ${token}`,
  }
});
const candidates = await candidatesResponse.json();
console.log('Assessment candidates:', candidates);
```
</CodeGroup>

## Common Issues & Solutions

<AccordionGroup>
  <Accordion title="Authentication Error (401)">
    **Problem**: Getting unauthorized errors  
    **Solution**: Check that your token is valid and properly formatted in the Authorization header:
    ```bash
    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
    ```
  </Accordion>
  
  <Accordion title="Organization Name Already Exists">
    **Problem**: Getting conflict error during registration  
    **Solution**: Organization names must be unique. Try a different name or check availability:
    ```bash
    curl -X GET https://api.skillment.in/api/organizations/validate/my-org-name
    ```
  </Accordion>
  
  <Accordion title="Question Validation Errors">
    **Problem**: Assessment creation fails with validation errors  
    **Solution**: Ensure all required fields are provided:
    - `question` text is not empty
    - `difficulty` is one of: "Easy", "Medium", "Hard"
    - For coding questions: at least one test case is provided
    - `marks` is a positive number
  </Accordion>
</AccordionGroup>
