## Overview

This quickstart guide will help you create your first assessment on Skillment in just a few minutes. By the end of this guide, you'll have:

- Set up your organization
- Created your first assessment
- Invited candidates
- Configured basic proctoring settings

<Note>
  You'll need a Skillment account to follow this guide. If you don't have one, [sign up here](https://app.skillment.com/signup).
</Note>

## Step 1: Organization Setup

After signing up, you'll be prompted to create your organization:

<Steps>
  <Step title="Choose Organization Name">
    Select a unique organization name that will be used in your URLs (e.g., `company-name.skillment.com`)
  </Step>
  <Step title="Add Organization Details">
    Fill in your organization's display name, type, and size for better customization
  </Step>
  <Step title="Upload Logo (Optional)">
    Upload your organization's logo for branded assessments and emails
  </Step>
</Steps>

```json Organization Creation Example
{
  "name": "acme-corp",
  "displayName": "ACME Corporation",
  "type": "corporate",
  "size": "50-200"
}
```

## Step 2: Create Your First Assessment

Navigate to the Assessments page and click "Create Assessment":

### Basic Information

<CodeGroup>

```json Assessment Configuration
{
  "title": "Frontend Developer Assessment",
  "description": "Comprehensive evaluation for React developers",
  "type": "technical",
  "duration": 60,
  "totalMarks": 100,
  "attemptLimit": 1
}
```

```json Question Example
{
  "question": "What is the virtual DOM in React?",
  "type": "multiple_choice",
  "options": [
    "A JavaScript representation of the real DOM",
    "A browser API for manipulating DOM",
    "A React component lifecycle method",
    "A state management library"
  ],
  "correctAnswer": 0,
  "marks": 5
}
```

</CodeGroup>

### Assessment Settings

Configure your assessment with these essential settings:

<Accordion title="Time Management">
  - **Duration**: Set the total time allowed (e.g., 60 minutes)
  - **Auto-submit**: Automatically submit when time expires
  - **Time warnings**: Alert candidates at 30, 15, and 5 minutes remaining
</Accordion>

<Accordion title="Question Settings">
  - **Randomize Questions**: Present questions in random order
  - **Randomize Options**: Shuffle multiple-choice options
  - **Show Results**: Display results immediately after submission
</Accordion>

<Accordion title="Security Settings">
  - **Proctoring**: Enable webcam monitoring
  - **Tab Switch Detection**: Monitor when candidates leave the assessment
  - **Full Screen Mode**: Force candidates to stay in full screen
</Accordion>

```javascript Multiple Choice Question
{
  "question": "Which hook is used for side effects in React?",
  "type": "multiple_choice",
  "options": [
    "useState",
    "useEffect",

```

## Step 4: Configure Proctoring

Enable security features to maintain assessment integrity:

<Tabs>
  <Tab title="Basic Proctoring">
    ```json Basic Settings
    {
      "webcamMonitoring": true,
      "tabSwitchDetection": true,
      "fullscreenMode": true,
      "rightClickDisable": true
    }
    ```
  </Tab>
  <Tab title="Advanced Proctoring">
    ```json Advanced Settings
    {
      "screenRecording": true,
      "environmentCheck": true,
      "idVerification": true,
      "suspiciousActivityThreshold": 3,
      "copyPasteDetection": true
    }
    ```
  </Tab>
</Tabs>

<Warning>
  Advanced proctoring features may require additional setup and candidate permissions for camera and screen access.
</Warning>

## Step 5: Invite Candidates

Once your assessment is ready, invite candidates:

### Bulk Invitation

1. **Upload CSV**: Prepare a CSV file with candidate information
2. **Email Templates**: Customize invitation emails with your branding
3. **Credentials**: Auto-generate secure login credentials

```csv CSV Format Example
name,email,phone,organization
John Doe,john@example.com,+1234567890,ACME Corp
Jane Smith,jane@example.com,+1234567891,Beta Inc
```

### Individual Invitation

For single candidates, use the individual invitation form:

```json Candidate Invitation
{
  "name": "John Doe",
  "email": "john@example.com",
  "assessmentId": "assessment_123",
  "customMessage": "Please complete this assessment by Friday."
}
```

## Common Issues

<AccordionGroup>
  <Accordion title="Candidates can't access the assessment">
    Check that:
    - Invitations were sent successfully
    - Assessment status is "published" not "draft"
    - Candidates are using the correct credentials
    - Assessment hasn't expired
  </Accordion>

  <Accordion title="Proctoring not working">
    Ensure:
    - Candidates have granted camera/microphone permissions
    - Browser supports WebRTC (Chrome, Firefox, Safari)
    - No browser extensions blocking camera access
    - Stable internet connection
  </Accordion>

  <Accordion title="Questions not displaying correctly">
    Verify:
    - Question content is properly formatted
    - Images are uploaded and accessible
    - Question types are correctly configured
    - Assessment is saved and published
  </Accordion>
</AccordionGroup>
