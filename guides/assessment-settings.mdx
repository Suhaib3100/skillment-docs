## Overview

Assessment settings control how participants experience your evaluation, from time limits and question randomization to security measures and result processing. Proper configuration ensures fair, secure, and effective assessments.

## Basic Assessment Configuration

### General Settings

<Steps>
  <Step title="Assessment Information">
    Set basic details:
    - **Title**: Clear, descriptive name
    - **Description**: Instructions for participants
    - **Instructions**: Pre-assessment guidelines
    - **Duration**: Total time allowed
    - **Language**: Interface language
  </Step>
  <Step title="Scheduling">
    Control when assessments are available:
    - **Start Date/Time**: When assessment becomes available
    - **End Date/Time**: Last moment for submission
    - **Time Zone**: Participant location consideration
    - **Grace Period**: Extra time for technical issues
  </Step>
  <Step title="Attempts">
    Limit assessment attempts:
    - **Number of Attempts**: Usually 1 for hiring
    - **Retake Policy**: Conditions for additional attempts
    - **Cooling Period**: Time between attempts
    - **Best Score**: Which attempt score to use
  </Step>
</Steps>

## Question Presentation

### Question Order

<AccordionGroup>
  <Accordion title="Sequential Order" icon="list-ol">
    **Fixed Question Sequence**
    - Questions appear in predefined order
    - Consistent experience for all candidates
    - Easier to design progressive difficulty
    - Simpler for analysis and comparison
  </Accordion>

  <Accordion title="Randomized Order" icon="random">
    **Shuffled Question Sequence**
    - Questions appear in random order
    - Reduces cheating through communication
    - Each candidate gets unique sequence
    - Maintains assessment integrity
  </Accordion>

  <Accordion title="Question Pools" icon="layer-group">
    **Dynamic Question Selection**
    - Select random questions from larger pool
    - Different candidates get different questions
    - Maintains consistent difficulty level
    - Prevents question sharing
  </Accordion>
</AccordionGroup>

### Answer Options

Configure multiple-choice question behavior:
- **Option Randomization**: Shuffle answer choices
- **Partial Credit**: Award points for partially correct answers
- **Negative Marking**: Deduct points for wrong answers
- **Skip Allowance**: Allow candidates to skip questions

## Time Management

### Time Limits

<Tabs>
  <Tab title="Overall Time Limit">
    **Total Assessment Duration**
    - Set maximum time for entire assessment
    - Include buffer time for technical issues
    - Consider question complexity and count
    - Default: 60-90 minutes for coding assessments
  </Tab>
  <Tab title="Per-Question Timing">
    **Individual Question Limits**
    - Set time limits for each question
    - Automatic progression after time expires
    - Different limits for different question types
    - Helpful for pacing and fairness
  </Tab>
  <Tab title="Section Timing">
    **Timed Sections**
    - Divide assessment into timed sections
    - Cannot return to previous sections
    - Useful for multi-part evaluations
    - Simulates real interview conditions
  </Tab>
</Tabs>

### Time Display Options

Control what timing information participants see:
- **Show Remaining Time**: Display countdown timer
- **Time Warnings**: Alerts at 75%, 90% completion
- **Question Time**: Show time spent on current question
- **Hide Time**: Remove time pressure for some assessments

## Security Settings

### Browser Controls

<AccordionGroup>
  <Accordion title="Full-Screen Mode" icon="expand">
    **Locked Browser Experience**
    - Force full-screen mode
    - Prevent window switching
    - Hide browser controls
    - Disable right-click context menu
  </Accordion>

  <Accordion title="Tab Management" icon="window-restore">
    **Tab Monitoring**
    - Detect tab switching attempts
    - Log focus loss events
    - Warning system for violations
    - Automatic submission on repeated violations
  </Accordion>

  <Accordion title="Copy/Paste Restrictions" icon="clipboard">
    **Content Protection**
    - Disable copy/paste functionality
    - Prevent text selection
    - Block clipboard access
    - Protect question confidentiality
  </Accordion>
</AccordionGroup>

### Device Restrictions

Control participant device usage:
- **Single Device**: One device login per candidate
- **IP Restrictions**: Limit access by location
- **Browser Requirements**: Specify supported browsers
- **Mobile Restrictions**: Allow/block mobile devices

### Monitoring Levels

<Tabs>
  <Tab title="Basic Monitoring">

### Violation Handling

Define responses to security violations:

1. **Warning System**: Progressive warnings before action
2. **Automatic Pause**: Temporary assessment suspension
3. **Forced Submission**: End assessment immediately
4. **Manual Review**: Flag for human evaluation
5. **Retake Option**: Allow one-time retake after violation

## Best Practices

<AccordionGroup>
  <Accordion title="Time Management" icon="clock">
    - Allow 10-15 minutes per coding question
    - Include 10% buffer time for technical issues
    - Test timing with internal candidates first
    - Provide clear time expectations in instructions
  </Accordion>

  <Accordion title="Security Balance" icon="shield-alt">
    - Match security level to assessment importance
    - Consider candidate experience vs. security needs
    - Provide clear security expectations upfront
    - Have technical support available during assessments
  </Accordion>

  <Accordion title="Accessibility" icon="universal-access">
    - Always enable basic accessibility features
    - Provide accommodation request process
    - Test with screen readers and assistive devices
    - Offer alternative formats when needed
  </Accordion>
</AccordionGroup>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Time Limit Problems" icon="hourglass">
    **Issue**: Candidates running out of time
    **Solutions**:
    - Review question difficulty vs. time allocation
    - Add buffer time for complex problems
    - Provide time management tips in instructions
    - Consider per-question time limits
  </Accordion>

  <Accordion title="Navigation Confusion" icon="compass">
    **Issue**: Candidates struggle with navigation
    **Solutions**:
    - Provide navigation tutorial or demo
    - Include clear instructions about navigation rules
    - Test navigation flow with sample users
    - Offer practice assessments
  </Accordion>

  <Accordion title="Security Violations" icon="exclamation-triangle">
    **Issue**: Legitimate candidates triggering security alerts
    **Solutions**:
    - Calibrate sensitivity appropriately
    - Provide clear security guidelines
    - Implement progressive warning system
    - Offer manual review for edge cases
  </Accordion>
</AccordionGroup>

<Note>
**Start simple and iterate.** Begin with basic settings and gradually add complexity based on your organization's needs and candidate feedback. Test all settings thoroughly before launching important assessments.
</Note>
