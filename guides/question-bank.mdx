## Overview

The question bank is your central repository for assessment questions. Build a comprehensive library of coding problems, multiple-choice questions, and custom evaluations that can be reused across multiple assessments.

## Creating Custom Questions

### Coding Question Creation

<Tabs>
  <Tab title="Question Setup">
    **Basic Information**
    ```yaml
    Title: "Array Sum Challenge"
    Difficulty: Intermediate
    Language: JavaScript, Python, Java
    Topic: Arrays, Algorithms
    Estimated Time: 15 minutes
    ```
  </Tab>
  <Tab title="Problem Statement">
    **Clear Problem Description**
    - Problem context and requirements
    - Input/output specifications
    - Constraints and edge cases
    - Example test cases
    - Hints for candidates (optional)
  </Tab>
  <Tab title="Test Cases">
    **Automated Validation**
    ```javascript
    // Public Test Cases (visible to candidates)
    testCase1: {
      input: [1, 2, 3, 4, 5],
      expectedOutput: 15,
      description: "Sum of positive numbers"
    }
    
    // Hidden Test Cases (for evaluation)
    testCase2: {
      input: [-1, -2, 3, 4],
      expectedOutput: 6,
      description: "Mixed positive and negative"
    }
    ```
  </Tab>
  <Tab title="Scoring">
    **Evaluation Criteria**
    - Correctness weight: 70%
    - Time complexity: 20%
    - Code quality: 10%
    - Bonus for optimal solution
  </Tab>
</Tabs>

### MCQ Question Creation

Build effective multiple-choice questions:

<Steps>
  <Step title="Write Clear Question">
    Create unambiguous question text with proper context
  </Step>
  <Step title="Design Answer Options">
    - 4-5 answer choices
    - Only one correct answer (or specify multiple)
    - Plausible distractors
    - Avoid "all of the above" options
  </Step>
  <Step title="Add Explanations">
    Provide detailed explanations for:
    - Why the correct answer is right
    - Why incorrect answers are wrong
    - Additional learning resources
  </Step>
  <Step title="Set Difficulty">
    Calibrate difficulty based on:
    - Expected candidate knowledge level
    - Time required to answer
    - Concept complexity
  </Step>
</Steps>

## Question Analytics

### Performance Insights

Track question effectiveness:
- **Success Rates**: Percentage of correct answers
- **Time Distribution**: How long candidates take
- **Common Mistakes**: Frequent error patterns
- **Difficulty Validation**: Actual vs. expected difficulty

### Usage Analytics

Monitor question utilization:
- **Usage Frequency**: Most/least used questions
- **Assessment Distribution**: Where questions appear
- **Team Preferences**: Department-specific usage patterns
- **Seasonal Trends**: Time-based usage patterns
